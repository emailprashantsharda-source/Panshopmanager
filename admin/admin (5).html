🩺 Orphan Shop Repair. Finds users whose users/{uid}/shops/{shopId} index entry exists but whose canonical /shops/{shopId} doc is missing. These users get "Missing or insufficient permissions" errors when their app tries to write bills, products, or any other shop data. The repair creates the missing canonical doc from the index metadata. Safe — only creates missing docs, never overwrites existing ones.

Scan & Repair Orphan Shops

For each user, walks their shop index, checks if each canonical shop doc exists, and (if not) creates it with ownerId, name, profession, createdAt filled in from the index entry. Idempotent — safe to re-run.
